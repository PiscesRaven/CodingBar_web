常見問題

Q1：為什麼是學習 Python 而不是其他學校常見的語法(VB, C++)呢？
不同於 Scratch 積木拖拉式的兒童教育用程式語言，Python 是一套學習門檻低，且在網路應用、人工智慧、大數據分析、區塊鏈等領域被廣泛運用的程式語言，C++對初學者而言學習門檻過高，而 VB 則是即將過時的程式語言，因此我們認為 Python 是現階段學習程式的最佳起點！
Q2：如果我的孩子有基礎的程式設計能力，可以從 Level 2 以後開始學習嗎？
可以的。但為了讓學生能順利銜接，我們會做些簡單的測試建議他適合的課程。
Q3：為什麼要花長達一兩年時間來學程式？不能一個暑假就學完嗎？
我們認為培養程式能力就跟學音樂一樣，是需要長期且大量的實作練習，單靠短期濃縮的課程固然可讓您的孩子考到高分，但也失去了培養核心素養的初衷；也因此，我們開發了CodingBar線上學習平台，透過循序漸進的課程規劃與線上師生互動平台，讓正值課業壓力逐漸加重的孩子可以每週撥一點時間，一點一滴地為程式基礎打底。
Q4:如何決定上實體或是線上課程呢？
我們建議高中以前零基礎的孩子，都能先上過一期初學者實體課程，建立基礎觀念後，依孩子的興趣和動機再來決定是否要上線上課程。線上學習雖然方便，但因缺乏同儕互動，一般學習完成率都偏低，尤其是年紀較小的孩子。所以我們會建議先透過實體課程建立基礎，後續再選擇線上課程進階學習。

你可以透過 CodingBar 系統直接提問，老師將在 24小時內回覆你的問題。
